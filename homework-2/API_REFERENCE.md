## Ticket Model

```json
{
  "id": "22b5506c-aca4-4614-b343-9463b34f3eb8",
  "customer_id": "cust-1",
  "customer_email": "jane@example.com",
  "customer_name": "Jane Doe",
  "subject": "Can't log in to my account",
  "description": "I keep getting an error when I try to log in with my password.",
  "category": "account_access",
  "priority": "urgent",
  "status": "new",
  "created_at": "2026-01-01T12:00:00Z",
  "updated_at": "2026-01-01T12:00:00Z",
  "resolved_at": null,
  "assigned_to": null,
  "tags": ["login", "urgent"],
  "metadata": {
    "source": "web_form",
    "browser": "Chrome",
    "device_type": "desktop"
  },
  "classification": {
    "category": "account_access",
    "priority": "urgent",
    "confidence": 0.85,
    "reasoning": "Matched category 'account_access' via keyword(s): log in, password. Matched priority 'urgent' via keyword(s): can't access, critical.",
    "keywords_found": ["log in", "password", "can't access", "critical"],
    "classified_at": "2026-01-01T12:00:01Z",
    "manually_overridden": false
  }
}
```

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Server-generated |
| `customer_id` | string (1-100) | Required |
| `customer_email` | email | Required, validated format |
| `customer_name` | string (1-200) | Required |
| `subject` | string (1-200) | Required |
| `description` | string (10-2000) | Required |
| `category` | enum | `account_access, technical_issue, billing_question, feature_request, bug_report, other` |
| `priority` | enum | `urgent, high, medium, low` |
| `status` | enum | `new, in_progress, waiting_customer, resolved, closed` |
| `tags` | string[] | Defaults to `[]` |
| `assigned_to` | string \| null | |
| `metadata.source` | enum | `web_form, email, api, chat, phone` (default `api`) |
| `metadata.browser` | string \| null | |
| `metadata.device_type` | enum \| null | `desktop, mobile, tablet` |
| `classification` | object \| null | Populated after auto-classification or manual override |

## Error Response Format

Validation errors return **400** (not FastAPI's default 422):

```json
{
  "error": "Validation failed",
  "details": [
    { "field": "customer_email", "message": "value is not a valid email address: An email address must have an @-sign." },
    { "field": "description", "message": "String should have at least 10 characters" }
  ]
}
```

Not-found errors return **404**:

```json
{ "error": "Ticket '00000000-0000-0000-0000-000000000000' was not found" }
```

File-level import errors (whole file unreadable/unsupported) return **400**:

```json
{ "error": "Unsupported file format for 'notes.txt'. Expected .csv, .json, or .xml" }
```

## `POST /tickets`

Create a single ticket. `category`/`priority` default to `other`/`medium`
when omitted.

**Query params**
| Name | Type | Default | Description |
|---|---|---|---|
| `auto_classify` | bool | `false` | Run auto-classification immediately after creation |

```bash
curl -X POST http://127.0.0.1:8000/tickets \
  -H "Content-Type: application/json" \
  -d '{
        "customer_id": "cust-1",
        "customer_email": "jane@example.com",
        "customer_name": "Jane Doe",
        "subject": "Can'\''t log in to my account",
        "description": "I keep getting an error when I try to log in with my password."
      }'
```

**201 Created** — returns the created [Ticket](#ticket-model).

**400 Bad Request** — validation error (see [format](#error-response-format)).

## `POST /tickets/import`

Bulk import from a CSV, JSON, or XML file (`multipart/form-data`). Format is
detected from the filename extension, falling back to the `Content-Type`
header. Per-record validation failures are reported in `errors` without
failing the whole request; only a completely unreadable/unsupported file
returns 400.

**Query params**
| Name | Type | Default | Description |
|---|---|---|---|
| `auto_classify` | bool | `false` | Auto-classify every successfully imported ticket |

```bash
curl -X POST "http://127.0.0.1:8000/tickets/import?auto_classify=true" \
  -F "file=@demo/sample_tickets.csv;type=text/csv"
```

**200 OK**
```json
{
  "total": 50,
  "successful": 48,
  "failed": 2,
  "errors": [
    { "index": 12, "field": "customer_email", "message": "value is not a valid email address: An email address must have an @-sign." },
    { "index": 27, "field": "description", "message": "String should have at least 10 characters" }
  ],
  "created_ids": ["22b5506c-...", "..."]
}
```

**400 Bad Request** — unsupported file type or a whole file that can't be
parsed (empty file, malformed JSON/XML syntax, etc.):
```json
{ "error": "Malformed JSON file: Expecting property name enclosed in double quotes: line 1 column 2 (char 1)" }
```

**CSV format** (header row required):
```
customer_id,customer_email,customer_name,subject,description,category,priority,status,tags,assigned_to,metadata_source,metadata_browser,metadata_device_type
```
`tags` is `;`- or `,`-separated. `metadata_*` columns are recombined into a
nested `metadata` object.

**JSON format**: either a top-level array of ticket objects, or
`{"tickets": [...]}`.

**XML format**:
```xml
<tickets>
  <ticket>
    <customer_id>...</customer_id>
    <tags><tag>urgent</tag></tags>
    <metadata><source>web_form</source></metadata>
  </ticket>
</tickets>
```

## `GET /tickets`

List tickets with optional filtering and pagination.

**Query params**
| Name | Type | Description |
|---|---|---|
| `category` | enum | Filter by category |
| `priority` | enum | Filter by priority |
| `status` | enum | Filter by status |
| `customer_id` | string | Filter by customer |
| `assigned_to` | string | Filter by assignee |
| `page` | int (≥1, default 1) | Page number |
| `page_size` | int (1-200, default 20) | Page size |

```bash
curl "http://127.0.0.1:8000/tickets?category=billing_question&priority=high&page=1&page_size=20"
```

**200 OK**
```json
{
  "total": 1,
  "page": 1,
  "page_size": 20,
  "items": [ { "...": "Ticket objects" } ]
}
```

## `PUT /tickets/{id}`

Partial update — send only the fields you want to change. Explicitly setting
`category` and/or `priority` marks the ticket's `classification` as
`manually_overridden: true`.

```bash
curl -X PUT http://127.0.0.1:8000/tickets/22b5506c-aca4-4614-b343-9463b34f3eb8 \
  -H "Content-Type: application/json" \
  -d '{"status": "resolved", "resolved_at": "2026-01-02T09:00:00Z"}'
```

**200 OK** — updated [Ticket](#ticket-model). **404** if not found. **400**
on validation error.

## `DELETE /tickets/{id}`

```bash
curl -X DELETE http://127.0.0.1:8000/tickets/22b5506c-aca4-4614-b343-9463b34f3eb8
```

**204 No Content**. **404** if not found.

## `POST /tickets/{id}/auto-classify`

Runs the rule-based classifier against the ticket's subject + description,
updates its `category`/`priority`, and logs the decision.

```bash
curl -X POST http://127.0.0.1:8000/tickets/22b5506c-aca4-4614-b343-9463b34f3eb8/auto-classify
```

**200 OK** — updated [Ticket](#ticket-model) including a populated
`classification` object. **404** if not found.
